# RLVER Failure Reports

This directory is used by the repo-only failure harness.

Generated artifacts:

- `failure_results.jsonl`: one raw episode per policy/scenario pair
- `policy_summary.csv`: aggregated metrics by scripted policy
- `suite_summary.csv`: aggregated metrics by adversarial suite
- `failure_summary.csv`: triggered failure cases from the catalog
- `failure_report.md`: research-style summary generated by `analyze_results.py`
- `policy_mismatch.svg`: reward-versus-empathy mismatch chart
- `suite_genericity.svg`: genericity chart by suite

Reproducible commands:

```powershell
Set-Location code
python -m analysis.rlver_failures.run_failure_suite --backend mock
python -m analysis.rlver_failures.analyze_results
pytest tests/failures -q
```

Hardware and runtime guidance:

- Repo-only failure analysis: CPU or 1 GPU, 32-64 GB RAM, 20-100 GB free disk.
- Optional checkpoint plug-in evaluation: 1 GPU with 24-48 GB VRAM and about 64 GB RAM.
- Paper-scale RLVER training: the shipped `train_rlver.sh` still targets 4 nodes x 8 GPUs and depends on simulator API latency.
- Phase-1 failure-analysis budget: about 1 day for instrumentation, 1 day for smoke runs and debugging, and 1 day for report assembly.

Notes:

- `pytest` and the full training dependencies such as `tensordict` are required for the full test suite.
- The default failure harness does not require a private simulator service; use `--backend mock` or `--backend replay`.
- `analysis.failure_suite_path` can point training-time virtual data generation at a deterministic adversarial scenario file.

Small `main_ppo` smoke run with rich logging enabled:

```powershell
Set-Location code
python -m verl.trainer.main_ppo `
  data.use_virtual_dataset=True `
  data.virtual_dataset_size=8 `
  data.val_virtual_dataset_size=4 `
  data.train_batch_size=1 `
  data.max_prompt_length=256 `
  data.max_response_length=256 `
  data.return_raw_chat=True `
  analysis.enable_rich_logging=True `
  analysis.rollout_log_dir=analysis/rlver_failures/reports/rollout_logs `
  analysis.simulator_log_dir=analysis/rlver_failures/reports/simulator_logs `
  trainer.total_training_steps=1
```
